# Homebrew

Declarative, idempotent Homebrew package management for macOS and Linux (via Home Manager).

## Overview

This module manages Homebrew packages through a declarative package list. Unlike manual Brewfile management, the package list is defined in Nix and generated automatically during `nh home switch`.

**Platform Support:**
- **macOS (Darwin)**: Primary target platform
- **Linux (generic)**: Supported via Linuxbrew
- **NixOS**: Not applicable (Homebrew is disabled on NixOS hosts)

## How It Works

The module generates `~/.Brewfile` from a Nix-defined package list:

1. **Package Definition**: Edit `config/home-manager/modules/applications/homebrew/homebrew-list.nix`
2. **Brewfile Generation**: The `brewfile.nix` module generates `~/.Brewfile` with proper syntax
3. **Activation Sync**: During `nh home switch`, the `homebrew.nix` activation script:
   - Checks if Homebrew is installed (graceful bail-out if not)
   - Reads `~/.Brewfile`
   - Installs missing packages
   - Removes top-level packages not in the list (preserves dependencies)
   - Optionally runs `brew upgrade` if `autoUpdate = true`

## Configuration

Enable Homebrew management in your host profile:

```nix
{ config, ... }:
{
  myHmStandalone.programs.homebrew = {
    enable = true;
    autoUpdate = false;  # Set to true to run 'brew upgrade' after installing
  };
}
```

## Package List Format

Edit `config/home-manager/modules/applications/homebrew/homebrew-list.nix`:

```nix
# @MODULON_SKIP
{ lib, hostKind, ... }:
{
  crossPlatform = [
    "brew:azure-cli"           # CLI tool (explicit)
    "ripgrep"                  # CLI tool (implicit brew)
    "cask:firefox"             # GUI application
    "tap:homebrew/cask-fonts"  # Custom tap
  ];

  darwinOnly = [
    "brew:maccy"               # macOS-specific tool
    "cask:raycast"             # macOS-specific GUI app
  ];

  linuxOnly = [
    "brew:gcc"                 # Linux-specific compiler
  ];
}
```

### Prefix Syntax

Packages use the `"type:name"` format. If no type is specified, `"brew"` is assumed.

| Prefix | Description | Example |
|--------|-------------|---------|
| `brew:formula` | CLI tools and formulae | `brew:ripgrep` |
| `cask:app` | macOS GUI applications | `cask:firefox` |
| `tap:owner/repo` | Custom repositories | `tap:homebrew/cask-fonts` |
| (none) | Defaults to brew | `ripgrep` → `brew:ripgrep` |

### Platform Lists

- **`crossPlatform`**: Packages installed on all platforms (Darwin and Linux)
- **`darwinOnly`**: Packages installed only on macOS
- **`linuxOnly`**: Packages installed only on Linux (not NixOS)

## Key Features

- **Declarative & idempotent** – Safe to run on every `nh home switch`
- **Cross-platform** – Works on both macOS and Linux via Homebrew
- **Prefix-based syntax** – Support for brew formulae, casks, and taps
- **Automatic tap handling** – Taps are extracted and added before packages
- **Top-level aware** – Uses `brew leaves` so only user-declared packages are uninstalled
- **Dependency friendly** – Leaves shared dependencies and lets `brew autoremove` handle orphans
- **Graceful degradation** – Skips synchronization if Homebrew is not installed
- **Cross-shell compatible** – Activation script uses POSIX shell with explicit tool resolution

## File Structure

```
config/home-manager/modules/applications/homebrew/
├── brewfile.nix        # Generates ~/.Brewfile from package lists
├── homebrew.nix        # Activation script for sync
└── homebrew-list.nix   # Package definitions (edit this)
```

## Performance Notes

- The sync runs during Home Manager activation (at `switch` time)
- Checks are cached in the activation script
- Empty or missing Brewfile is handled gracefully
- No-op if Homebrew is not installed on the system

## Troubleshooting

### Package installation fails

- Check that Homebrew is installed and reachable at `/opt/homebrew/bin/brew` (macOS) or `/home/linuxbrew/.linuxbrew/bin/brew` (Linux)
- The module prepends common Homebrew paths to PATH automatically
- Ensure you have network connectivity
- Look for error messages in the activation output

### Brewfile not found

- The module gracefully skips if `~/.Brewfile` doesn't exist
- This file is auto-generated by `brewfile.nix` from your `homebrew-list.nix`
- Verify `programs.homebrew.enable = true` in your profile

### Module not loading

- Ensure `homebrew-list.nix` has `# @MODULON_SKIP` at the top
- This prevents the module loader from trying to import it as a Home Manager module
